## Executive Summary

Every generation attempt from `atelier.autonomous-agent.dev` fails silently with
**"Pipeline error: Failed to fetch"**. The root cause is a stale Firebase Hosting
deployment: the live Content Security Policy `connect-src` directive does not include
`https://*.run.app`, so the browser blocks every POST to the Cloud Run API endpoint
before any bytes leave the machine. The fix is already committed to `firebase.json`
but has not been deployed. A one-command `firebase deploy --only hosting` resolves
all generation failures and restores all API-backed toolbar buttons.

## Root Cause Analysis

### Primary Bug — CSP `connect-src` missing `*.run.app` (stale deployment)

**What the live site served** (captured from response headers of event #10):

```
Content-Security-Policy:
  default-src 'self';
  ...
  connect-src 'self'
    https://*.googleapis.com
    https://*.firebaseio.com
    wss://*.firebaseio.com
    https://identitytoolkit.googleapis.com
    https://accounts.google.com
    https://www.google.com/recaptcha/
    https://recaptcha.net/;
  ...
```

`https://*.run.app` is **absent**. The Cloud Run API endpoint
`atelier-api-staging-537337457799.us-central1.run.app` does not match any directive,
so the browser refuses the fetch.

**What the current `firebase.json` specifies** (both hosting entries, lines 41 and 141):

```
connect-src 'self'
  https://*.googleapis.com
  https://*.firebaseio.com
  wss://*.firebaseio.com
  https://identitytoolkit.googleapis.com
  https://accounts.google.com
  https://www.google.com/recaptcha/
  https://recaptcha.net/
  https://*.run.app               ← present in code, absent in deployment
  https://*.autonomous-agent.dev; ← present in code, absent in deployment
```

The fix was committed to `firebase.json` (commit history: `12166ce fix(dashboards):
execute R11 sprint — CSP, fonts, XSS hardening`) but **Firebase Hosting was not
redeployed** with that commit's configuration.

**API URL source** — `atelier-dashboard/cloudbuild.yaml:31`:

```yaml
_API_URL: 'https://atelier-api-staging-537337457799.us-central1.run.app'
```

This is baked into the Next.js image at build time as `NEXT_PUBLIC_API_URL`. The
browser therefore makes direct cross-origin requests to `*.run.app`. No intermediate
proxy is involved — the CSP must explicitly allow the target origin.

**CORS is NOT the issue** — `atelier-deploy/terraform/main.tf` sets:

```hcl
env {
  name  = "ATELIER_DASHBOARD_ORIGIN"
  value = "https://atelier.autonomous-agent.dev,https://atelier-build-2026.web.app"
}
```

The Cloud Run service's `CORSMiddleware` correctly allows `atelier.autonomous-agent.dev`.
Once CSP unblocks the request, CORS will pass without changes.

### Secondary Issues (non-blockers)

**Non-functional toolbar buttons** (model swap, recording, tab navigation):
All buttons that invoke the generation API call the same `NEXT_PUBLIC_API_URL`
endpoint. They fail for the same CSP reason — not independent bugs. Resolved by
the same deploy.

**Irrelevant generated surfaces** (pricing table, testimonials, hero sections):
The reporter received a completed generation (presumably from a prior run or cache)
that included surfaces irrelevant to a Linear-style project-management dashboard.
This is a planner surface-scope quality issue, not an infrastructure bug. The pipeline
never successfully completed a fresh run during this session to validate current
behavior. Investigate separately after the CSP fix restores generation.

**tldraw font ERR_CACHE_MISS** (events 1–4):
These are benign — the browser's resource-timing API emits a "miss" event on the
first speculative fetch before the CDN request completes. Events 5–8 show the same
URLs resolving successfully at 200 ~100ms later. No action needed.

## Fix

### Immediate — One command

```bash
# From the repo root
firebase deploy --only hosting:atelier-build-2026
```

This redeploys Firebase Hosting with the corrected CSP already in `firebase.json`
(both the `atelier-build-2026` site entry and the `dashboard` target). After deploy,
the live CSP will include `https://*.run.app`, unblocking all browser-to-API fetches.

### Verification

```bash
# Confirm live CSP includes *.run.app after deploy
curl -sI https://atelier.autonomous-agent.dev/ \
  | grep -i content-security-policy \
  | grep -o 'run\.app'
# Expected output: run.app
```

Then open https://atelier.autonomous-agent.dev/studio, type any brief, click Run —
the generation stream should initiate and SSE events should begin flowing within 2–3s.

### Long-term consideration (optional)

The current architecture makes the browser call Cloud Run directly. Firebase Hosting
already has an `/api/**` rewrite to `atelier-api-staging`. If `NEXT_PUBLIC_API_URL`
were changed to `""` (empty, so `getApiUrl()` falls back to `""` and all calls go to
`/api/v1/generate/stream`), the requests would be same-origin and bypass CSP and CORS
entirely. However, SSE streaming through Firebase Hosting CDN may introduce buffering
or timeout issues that break the long-lived stream. The direct Cloud Run URL + CSP
allowlist is the safer architecture for streaming — just ensure the CSP stays deployed.
