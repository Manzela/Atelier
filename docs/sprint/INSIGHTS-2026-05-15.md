## Operational lessons (apply on every session)

### 1. Pre-commit prettier auto-fix recursion — solved pattern

**Symptom**: `git commit` triggers prettier → prettier reformats files → hook fails with "files were modified by hook" → pre-commit rolls back the stash → commit silently reverts.

**Root cause**: Pre-commit's `pre-commit-hooks` rollback behavior when a fixer modifies staged content.

**Fix (proven this session)**: Stage all unstaged changes (including prettier auto-fixes from prior failed attempts) BEFORE re-running `git commit`. Second pre-commit run sees clean files and passes.

```bash
# When you hit the recursion:
git add -A                    # stage prettier's auto-fixes from the failed run
git commit -m "..."           # second pre-commit run sees clean files → passes
```

**Anti-pattern**: Do NOT use `--no-verify`. CLAUDE.md hard rule. The fix above is the legitimate path.

### 2. Branch protection + admin-merge for green Dependabot

**Setting**: Branch protection on `main` requires PR review (`enforce_admins: false` to allow legitimate admin operations).

**Pattern for green security PRs during sprint bootstrap**:

```bash
gh pr merge <N> --squash --admin   # only when status checks all green + dependabot-vetted
```

PR #20 (GHA bumps, all checks SUCCESS) was admin-merged this session. Document admin-merge rationale in commit if not obvious.

### 3. Release-please needs explicit GHA permissions

**Symptom**: Release workflow fails with "GitHub Actions is not permitted to create or approve pull requests".

**One-time fix** (already applied to this repo):

```bash
gh api -X PUT /repos/Manzela/atelier/actions/permissions/workflow \
  -f default_workflow_permissions=write \
  -F can_approve_pull_request_reviews=true
```

If you set up release-please on a new repo, run this immediately after `gh repo create`.

### 4. Rebase before push when remote moved (admin-merge upstream)

After admin-merging a PR into `main`, the local branch is behind. Pattern:

```bash
git pull --rebase origin main   # fast-forward over the merged PR
git push origin main            # now succeeds
```
